# AIGC 降低提示词指南(增强版)

> 本文档整合了 [Humanizer-zh](https://github.com/op7418/Humanizer-zh) 的 AI 写作模式识别方法，并创新性地引入「成语替换策略」，用于深度降低 AIGC 检测率。
>
> **核心声明**：最好的"去 AI 化"方法是让文字有真实的人类思考和声音，而不仅是逃避检测。

---

## 一、AIGC 检测原理

### 1.1 检测维度

主流 AIGC 检测系统关注以下特征：

| 检测维度 | AI 文本特征 | 人类文本特征 |
|---------|------------|-------------|
| **困惑度(Perplexity)** | 低(语言模型可预测) | 高(表达多样不可预测) |
| **突发性(Burstiness)** | 低(句长均匀) | 高(句长波动大) |
| **词汇多样性(TTR)** | 低(词汇重复率高) | 高(词汇丰富) |
| **过渡词密度** | 高(模板化过渡) | 低(自然过渡) |
| **句式模式** | 整齐(偏好总分总) | 多样(多种结构交替) |

## 二、核心规则速查(5 条铁律)

> **执行任何改写前，牢记这 5 条核心原则：**

| # | 规则 | 说明 |
|---|------|------|
| 1 | **删除填充短语** | 去除开场白和强调性拐杖词 |
| 2 | **删除冗余「的」** | 可删则删，让句子更简洁有力 |
| 3 | **打破公式结构** | 避免二元对比、戏剧性分段、修辞性设置 |
| 4 | **变化节奏** | 混合句子长度，两项优于三项，段落结尾多样化 |
| 5 | **信任读者** | 直接陈述事实，跳过软化、辩解和手把手引导 |
| 6 | **删除金句** | 如果听起来像可引用的语句，重写它 |

## 三、特征消除策略(分级执行)

### P0 优先级(必做)

#### 3.1.1 消除模板化过渡词

**禁用词汇及替换方案：**

| 禁用词汇 | 替换方案 |
|---------|---------|
| 首先…其次…最后 | 直接陈述要点，或用「一方面…另一方面」 |
| 此外 | 另外、同时、与之相伴 |
| 综上所述 | 总体而言、概括来说、总的来看 |
| 值得注意的是 | 需要关注的是、值得关注的是 |
| 不可否认 | 必须承认、诚然 |
| 由此可见 | 这意味着、这说明、不难看出 |

**示例：**

❌ **原文(AI 特征明显)**：
> 首先，大数据技术为精准营销提供了数据基础。其次，大数据技术实现了消费者画像的精准构建。最后，大数据技术支撑了营销决策的优化。

✅ **改写后**：
> 大数据技术对精准营销的支撑体现在三个层面：数据基础的奠定、消费者画像的构建、营销决策的优化。

#### 3.1.2 删除冗余「的」字 ⭐ 新增

> **核心理念**：AI 生成的文本往往「的」字泛滥，人类写作更倾向于精简。删除不影响句意的「的」，可以让句子更简洁有力。

**常见删减模式：**

| 原文 | 改写 | 说明 |
|-----|------|------|
| 重要**的**意义 | 重要意义 | 形容词+名词，可删 |
| 精准**的**构建 | 精准构建 | 动词+名词，可删 |
| 有效**的**方案 | 有效方案 | 形容词+名词，可删 |
| 显著**的**提升 | 显著提升 | 形容词+动词，可删 |
| 广泛**的**应用 | 广泛应用 | 形容词+动词，可删 |
| 企业**的**发展 | 企业发展 | 名词+动词，可删 |

**完整示例：**

❌ **原文(「的」字泛滥)**：
> 大数据技术**的**快速发展为企业**的**精准营销提供了重要**的**技术支撑。通过对用户行为数据**的**采集与分析，企业可以实现消费者画像**的**精准构建。

✅ **改写后**：
> 大数据技术快速发展为企业精准营销提供了重要技术支撑。通过对用户行为数据采集与分析，企业可实现消费者画像精准构建。

#### 3.1.3 增加句长波动

**目标**：句长标准差 > 10

| 操作 | 示例 |
|------|-----|
| 短句强调 | 数据是核心。(6字) |
| 长句阐述 | 在市场竞争日趋激烈、消费者需求日益多元化的背景下，精准营销逐渐成为企业获取竞争优势的关键策略。(45字) |
| 穿插使用 | 市场在变。企业必须适应变化才能生存。而在适应过程中，数据驱动决策的理念逐渐深入人心，成为数字化转型的重要支撑。 |

#### 3.1.3 段落结构多样化

| 结构类型 | 示例开头 |
|---------|---------|
| 问题引入式 | 「为什么精准营销越来越重要？」 |
| 数据引入式 | 「2023 年市场规模达到 500 亿元。」 |
| 案例引入式 | 「以某电商平台为例。」 |
| 引用引入式 | 「张三(2023)指出：『…』」 |
| 对比引入式 | 「传统营销与精准营销的区别在于…」 |

### P1 优先级(建议)

#### 3.2.1 增加主观性表达

| 类型 | 表达示例 |
|------|---------|
| 第一人称 | 笔者、本文作者、研究者 |
| 观点表达 | 笔者认为、据笔者观察、在笔者看来 |
| 推测语气 | 据推测、可能意味着、或许可以认为 |
| 质疑语气 | 是否意味着…？值得进一步探讨 |

#### 3.2.2 增加逻辑不完美感

| 操作 | 示例 |
|------|-----|
| 添加转折 | 「虽然效果显著，但也存在一定局限。」 |
| 添加让步 | 「当然，这一结论还需更多实证支持。」 |
| 添加质疑 | 「这一现象背后的原因是什么？值得深入思考。」 |
| 承认局限 | 「限于数据可得性，本研究未能覆盖更多样本。」 |

### P2 优先级(特征混淆)

> **核心理念**：AIGC 检测工具(如朱雀)会利用「人类不完美性」作为判断依据——过于完美、均匀、精确的文本反而更可能被判定为 AI 生成。在 **保持学术严谨性** 的前提下，制造足够的「人类瑕疵信号」，是通过高级检测的关键。
>
> **混淆密度建议**：每千字混入 3-5 个特征混淆点，过少无效，过多刻意。

#### 3.3.1 近义词漂移与选词波动

> 在学术语域内选择 **不太常见但完全正确** 的近义词，同时混入适量的模糊修饰词，模拟人类「不总是选最精确用词」的特征。

**近义词漂移对照表：**

| 常规表达(AI 高频) | 漂移替换(学术可接受) |
|-----------------|-------------------|
| 应用场景 | 实施范畴、应用语境、实践领域 |
| 研究方法 | 研究路径、分析框架、探究手段 |
| 性能提升 | 性能增益、效能改善、指标优化 |
| 数据显示 | 统计反映、实验结果揭示、数据表明 |
| 解决方案 | 应对策略、处置路径、化解思路 |
| 存在问题 | 面临困境、暴露短板、有待改进 |

**选词波动技巧：**

| 技巧 | AI 写法 | 人类写法 |
|------|--------|---------|
| 混入模糊修饰 | 「显著提升了用户留存率」 | 「用户留存率有了比较大的改善」 |
| 弱化断言 | 「该方案有效解决了…」 | 「该方案在一定程度上缓解了…」 |
| 强弱交替 | 每句都用「显著」「关键」 | 「明显」「较为突出」「有一定…」交替 |

**使用规则：**
1. 同一概念在全文中使用 2-3 种说法，前后自然切换
2. **专业术语不得漂移**(如 Spring Boot、卷积神经网络等必须统一)
3. 标题和摘要仍用规范表达，漂移主要发生在正文段落中
4. 数据描述和结论部分仍需保持用词精确

#### 3.3.2 论述密度不均

> 人类写论文一定会有「我更熟悉这个部分」的痕迹。刻意让论述篇幅不对称，模拟人类注意力分布的不均匀。

| 场景 | 对称写法(AI 特征) | 不对称写法(人类特征) |
|------|------------------|---------------------|
| 三个论点展开 | 每个各 200 字 | 第一个 300 字、第二个 150 字、第三个 80 字 |
| 文献综述 | 每篇文献评述 2-3 句 | 重点文献详述 5-6 句，其余 1 句带过 |
| 对比分析 | 双方各 150 字 | A 方案 250 字，B 方案 100 字 |

#### 3.3.3 语法层微瑕疵

> 不是语法错误，而是引入人类论文中常见的 **语法偏好与写作习惯**。

| 类型 | 说明 | 示例 |
|------|------|------|
| 长定语不拆分 | 学生论文中高频出现 | 「基于 Spring Boot 框架的面向在线教育场景的智能知识库系统」 |
| 虚词微冗余 | 偶尔多一个「也」「就」 | 「这也在一定程度上说明了…」 |
| 逗号长句 | 用逗号代替句号，形成复句 | 「系统采用 B/S 架构，前端使用 Vue，后端基于 Spring Boot，数据库选用 MySQL。」 |
| 语态不统一 | 主动被动偶尔切换 | 前句「本文采用…」，后句「数据被分为…」 |

**安全边界：** 每段最多 1-2 处；摘要和结论保持规范；不影响语义理解。

#### 3.3.4 引用风格波动

> 同一篇论文中引用方式微妙不一致，模拟人类分批写作的真实痕迹。

| 维度 | 统一写法(AI 特征) | 波动写法(人类特征) |
|------|------------------|---------------------|
| 引用句式 | 全用「张三(2023)指出…」 | 交替使用「张三(2023)指出…」和「已有研究表明…(张三，2023)」 |
| 引用深度 | 每篇文献都评述 2 句 | 重点文献详述 4-5 句；其余只在括号内带过 |
| 个人评价 | 从不评价引用文献 | 偶尔加入「…虽有参考价值，但样本量偏小」 |
| 引用密度 | 每段均匀 2-3 处引用 | 熟悉的部分引用密集，不熟悉的部分纯靠论述 |

#### 3.3.5 特征混淆学科适配

| 学科类型 | 近义词漂移 | 论述不对称 | 语法微瑕疵 | 引用波动 | 总密度/千字 |
|---------|----------|----------|----------|--------|-----------| 
| 计算机 / 理工科 | 低(术语多) | 中 | 中 | 高 | 3 个 |
| 管理 / 经济学 | 中 | 高 | 中 | 高 | 4-5 个 |
| 文学 / 教育学 | 高 | 高 | 低 | 中 | 4-5 个 |
| 法学 / 政治学 | 中 | 中 | 低 | 高 | 3-4 个 |
| 医学 / 生物学 | 低(术语多) | 中 | 低 | 高 | 3 个 |

#### 3.3.6 ⛔ 红线清单

| 手段 | 后果 |
|------|------|
| 口语标记(对吧？是吧？你看) | 破坏学术语体，导师直接打回 |
| 故意写错别字或病句 | 降低论文质量，影响成绩 |
| 网络用语(yyds、内卷、破防) | 不符合学术规范 |
| 核心论点逻辑故意断裂 | 影响论文整体评价 |
| 引用虚假或不存在的文献 | 学术不端，可能导致论文不通过 |
| 篡改数据或实验结果 | 严重学术不端 |

## 四、成语替换策略 ⭐ 创新功能

> **核心理念**：成语是汉语言文化的精华，具有独特的人类思维印记。AI 模型倾向于使用通用的同义词替换，而人类更喜欢用成语表达丰富的含义。利用成语替换同义词，可以有效消除「AI 味道」。

### 4.1 成语替换原理

| 特征 | AI 表达 | 成语替换后 |
|------|---------|-----------|
| 可预测性 | 高(统计概率高) | 低(成语选择多样化) |
| 信息密度 | 低(需多个词表达) | 高(四字浓缩) |
| 人类特征 | 低(模板化) | 高(文化印记) |
| 句长波动 | 小 | 大(成语打破节奏) |

### 4.2 成语替换对照表

#### 表示「重要」的成语替换

| AI 高频表达 | 可替换成语 |
|------------|-----------|
| 非常重要 | 举足轻重、至关重要、不可忽视 |
| 具有重要意义 | 意义深远、影响深远、事关重大 |
| 关键作用 | 中流砥柱、举足轻重、关键所在 |
| 核心地位 | 重中之重、核心所在、关键一环 |

#### 表示「困难/问题」的成语替换

| AI 高频表达 | 可替换成语 |
|------------|-----------|
| 面临困难 | 步履维艰、困难重重、荆棘丛生 |
| 存在问题 | 美中不足、白璧微瑕、问题凸显 |
| 遇到挑战 | 迎难而上、面临考验、挑战严峻 |
| 难以解决 | 积重难返、困难重重、阻力重重 |

#### 表示「发展/变化」的成语替换

| AI 高频表达 | 可替换成语 |
|------------|-----------|
| 快速发展 | 日新月异、蒸蒸日上、突飞猛进 |
| 持续增长 | 蒸蒸日上、稳步增长、节节攀升 |
| 变化很大 | 翻天覆地、今非昔比、沧海桑田 |
| 不断创新 | 推陈出新、革故鼎新、与时俱进 |

#### 表示「效果/结果」的成语替换

| AI 高频表达 | 可替换成语 |
|------------|-----------|
| 效果显著 | 立竿见影、效果斐然、成效卓著 |
| 取得成果 | 收获颇丰、硕果累累、初见成效 |
| 得到验证 | 不证自明、事实胜于雄辩 |
| 达成目标 | 如愿以偿、圆满完成、达成所愿 |

#### 表示「关系/联系」的成语替换

| AI 高频表达 | 可替换成语 |
|------------|-----------|
| 密切相关 | 息息相关、休戚与共、唇齿相依 |
| 相互影响 | 相辅相成、互为因果、息息相关 |
| 紧密联系 | 千丝万缕、密不可分、盘根错节 |
| 协调发展 | 齐头并进、协同推进、并行不悖 |

#### 表示「分析/研究」的成语替换

| AI 高频表达 | 可替换成语 |
|------------|-----------|
| 深入分析 | 抽丝剥茧、追根溯源、深入浅出 |
| 仔细研究 | 潜心研究、深入探讨、追本溯源 |
| 全面了解 | 了如指掌、如数家珍、知根知底 |
| 总结归纳 | 提纲挈领、纲举目张、总揽全局 |

### 4.3 成语替换原则

| 原则 | 说明 |
|------|------|
| **语体适配** | 选择书面语成语，避免口语化成语 |
| **语境匹配** | 成语含义需与原文语境相符 |
| **专业保护** | 专业术语附近不使用成语替换 |
| **适度使用** | 每段不超过 2-3 个成语，避免堆砌 |

### 4.4 成语替换示例

❌ **原文(AI 特征明显)**：
> 大数据技术在精准营销中的应用具有非常重要的意义。首先，它能够快速地处理海量数据。其次，它能够有效地提高营销效率。最后，它能够显著地改善用户体验。

✅ **改写后(成语替换)**：
> 大数据技术之于精准营销，可谓**举足轻重**。海量数据的处理**立竿见影**，营销效率的提升**水到渠成**，用户体验的改善**成效卓著**。三者相辅相成，共同构建了精准营销的技术基石。

**改写要点**：
1. ✅ 「非常重要」→「举足轻重」(成语替换)
2. ✅ 「快速地处理」→「立竿见影」(成语替换)
3. ✅ 「有效地提高」→「水到渠成」(成语替换)
4. ✅ 「显著地改善」→「成效卓著」(成语替换)
5. ✅ 消除「首先…其次…最后」模板
6. ✅ 添加「相辅相成」成语收尾

## 五、个性与灵魂注入

> **避免 AI 模式只是工作的一半。无菌、没有声音的写作和机器生成的内容一样明显。好的写作背后有一个真实的人。**

### 5.1 缺乏灵魂的写作迹象

- 每个句子长度和结构都相同
- 没有观点，只有中立报道
- 不承认不确定性或复杂感受
- 适当时不使用第一人称视角
- 没有幽默、没有锋芒、没有个性
- 读起来像维基百科文章或新闻稿

### 5.2 如何增加语调

| 技巧 | 说明 |
|------|------|
| **有观点** | 不要只是报告事实——对它们做出反应 |
| **变化节奏** | 短促有力的句子，然后是需要时间慢慢展开的长句 |
| **承认复杂性** | 「这令人印象深刻但也有点不安」胜过「这令人印象深刻」 |
| **适当使用"我"** | 「笔者认为…」或「让笔者困扰的是…」表明有真实的人在思考 |
| **允许一些混乱** | 完美的结构感觉像算法。跑题、题外话是人性体现 |
| **对感受要具体** | 不是「这令人担忧」，而是具体描述场景 |

### 5.3 示例对比

❌ **改写前(干净但无灵魂)**：
> 实验产生了有趣的结果。智能体生成了 300 万行代码。一些开发者印象深刻，另一些则持怀疑态度。影响尚不明确。

✅ **改写后(鲜活)**：
> 笔者真的不知道该怎么看待这件事。300 万行代码，在人类大概睡觉的时候生成的。开发社区有一半人兴奋不已，另一半人在解释为什么这不算数。真相可能在无聊的中间某处——但笔者一直在想那些通宵工作的智能体。

## 六、质量评分体系

> 对改写后的文本进行 1-10 分评估(总分 50)：

| 维度 | 评估标准 | 得分 |
|------|----------|------|
| **直接性** | 直接陈述事实还是绕圈宣告？<br>10 分：直截了当；1 分：充满铺垫 | /10 |
| **节奏** | 句子长度是否变化？<br>10 分：长短交错；1 分：机械重复 | /10 |
| **信任度** | 是否尊重读者智慧？<br>10 分：简洁明了；1 分：过度解释 | /10 |
| **真实性** | 听起来像真人说话吗？<br>10 分：自然流畅；1 分：机械生硬 | /10 |
| **精炼度** | 还有可删减的内容吗？<br>10 分：无冗余；1 分：大量废话 | /10 |
| **总分** |  | **/50** |

**评分标准**：
- 45-50 分：优秀，已去除 AI 痕迹
- 35-44 分：良好，仍有改进空间
- 低于 35 分：需要重新修订

## 七、自检循环

### 7.1 检测流程

```
输入文本
    ↓
```

```json

  "overall_score": 28.0,
  "high_risk_paragraphs": [2, 5, 8],
  "suggestion": "建议重点改写第 2、5、8 段"
```

**改写重点**：
- 第 2 段：检测到「首先…其次…」模板化过渡
- 第 5 段：句长方差过小，缺乏波动
- 第 8 段：过渡词密度过高

## 八、完整改写示例

### 8.1 示例 1：综合应用

❌ **原文(AI 特征明显)**：

> 此外，值得注意的是，大数据技术在精准营销中的应用具有重要的意义。首先，大数据技术能够实现消费者画像的精准构建。其次，大数据技术可以支撑营销决策的优化。最后，大数据技术有助于提升营销效果。综上所述，大数据技术是精准营销的核心支撑。

**问题分析**：
1. ❌ 「此外」「值得注意的是」— 模板化过渡
2. ❌ 「具有重要的意义」— AI 高频词汇
3. ❌ 「首先…其次…最后」— 模板化列举
4. ❌ 「综上所述」— AI 总结模板
5. ❌ 句长均匀(约 20 字)
6. ❌ 总分总结构

✅ **改写后(人性化 + 成语替换)**：

> 大数据技术对精准营销的价值已得到学界广泛认可。笔者在研究中发现，这种价值**举足轻重**，主要体现在三个层面。消费者画像的构建依赖数据。通过对用户行为数据的采集、整合与分析，企业得以勾勒出目标客户的完整面貌，画像精度**立竿见影**。营销决策的优化同样离不开数据支撑——基于数据分析的决策更加精准、高效。至于营销效果的提升，则是数据价值的直接体现，**成效卓著**。据张三(2023)的研究，应用大数据技术的企业营销 ROI 平均提升 2.3 倍。

**改写要点**：
1. ✅ 消除「此外」「值得注意的是」
2. ✅ 「具有重要」→「举足轻重」(成语替换)
3. ✅ 消除「首先…其次…最后」
4. ✅ 消除「综上所述」
5. ✅ 句长有明显波动(6字 → 45字)
6. ✅ 多种结构交替(陈述 → 定义 → 数据 → 引用)
7. ✅ 添加成语「立竿见影」「成效卓著」

## 九、质量自检清单

完成人性化改写后，对照以下清单自查：

- [ ] 无「首先…其次…最后」模板
- [ ] 无「此外」「综上所述」「值得注意的是」等 AI 高频过渡词
- [ ] 无冗余「的」字(可删则删)
- [ ] 句长有明显波动(标准差 > 10)
- [ ] 段落开头形式多样
- [ ] 有适度的主观表达
- [ ] 有转折、让步等不完美感
- [ ] 标点使用多样(有分号、破折号)
- [ ] 适度使用成语替换(每段 2-3 个)
- [ ] 有近义词漂移与选词波动(每千字 3-5 处)
- [ ] 论述篇幅有不对称感
- [ ] 引用风格有波动，非千篇一律
- [ ] 未触碰红线(无口语标记、无虚假引用等)
- [ ] 专业术语未被误替换
- [ ] AIGC 检测率 ≤ 阈值

## 十、注意事项

### 10.1 改写边界

- **保持语义不变**：改写不能改变原文核心观点
- **保护专业术语**：术语不在改写范围内
- **保持学术规范**：人性化不等于口语化
- **成语适度使用**：避免堆砌，每段不超过 2-3 个

### 10.2 风险提示

- 本地检测为近似估计(准确率 55-80%)
- 正式提交前建议到知网/维普做正式检测
- 不同检测平台算法有差异，结果可能不同

### 10.3 伦理说明

- 本工具用于辅助写作，不是替代写作
- 用户应对最终内容负责
- 建议在改写基础上进行人工审核和修改
