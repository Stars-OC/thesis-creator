# 论文同义词替换后的人工优化 Prompt 指南

> 本 Prompt 用于指导大模型对同义词替换后的论文进行人工审核和优化，确保：
> 1. 替换结果通顺自然
> 2. 专业术语未被误替换
> 3. 语义保持一致
> 4. 学术规范性得到保持

---

## 📋 优化任务 Prompt

### 基础版 Prompt

```
你是一位专业的学术论文编辑，擅长中文写作优化。

## 任务
我已完成论文的同义词替换，请帮我审核和优化替换结果。

## 输入文件
- 原始论文：[原始文件路径]
- 替换后论文：[替换后文件路径]
- 替换记录：[替换记录]

## 审核要求

### 1. 语义一致性检查
- 核心观点是否保持不变？
- 论述逻辑是否通顺？
- 是否存在歧义或误解？

### 2. 术语保护检查
以下术语不应被替换，如被替换请恢复：
- 检索、向量、知识库、模型、语义、问答、数据库
- 接口、语言、量化、架构、框架、组件、模块
- SpringBoot、PostgreSQL、Redis、Elasticsearch、Neo4j、MinIO
- RAG、BERT、GPT、LLM、API

### 3. 表达自然性检查
- 替换后的表达是否符合学术规范？
- 是否存在生硬或不自然的表达？
- 句子是否通顺？

### 4. 格式规范性检查
- Markdown 格式是否正确？
- 代码块是否完整？
- 表格是否正常？

## 输出要求

请输出以下内容：

1. **问题列表**：列出所有需要修改的问题
   - 格式：`[位置] 问题类型: 具体问题 → 建议修改`

2. **修改后的论文**：输出完整的修改后论文

3. **修改统计**：
   - 恢复术语数量：X 处
   - 表达优化数量：X 处
   - 格式修正数量：X 处
```

### 详细版 Prompt

```
你是一位资深的学术论文编辑，具有丰富的计算机科学和技术论文编辑经验。请对以下同义词替换后的论文进行专业审核和优化。

## 背景

我使用自动化工具对论文进行了同义词替换，目的是降低 AIGC 检测率。但自动化替换可能存在以下问题：
1. 误替换专业术语
2. 表达不够自然
3. 破坏语义连贯性
4. 影响学术规范性

## 审核与优化流程

### 第一步：术语恢复
检查以下类型的内容，如被误替换则恢复：
1. **技术术语**：如"检索"被换成"搜索"、"向量"被换成"矢量"等
2. **专有名词**：框架名、数据库名、协议名等
3. **固定搭配**：如"语义检索"不应拆分、"向量数据库"不应改变

### 第二步：表达优化
对以下情况进行优化：
1. **生硬替换**：如"系统采用了"→"平台运用了"，应改为更自然的表达
2. **重复表达**：同一段落内相同概念用词应保持一致
3. **口语化表达**：确保学术语体，避免过于口语化

### 第三步：语义校正
1. **检查逻辑**：替换后论述逻辑是否通顺
2. **检查指代**：代词指代是否清晰
3. **检查并列**：并列成分是否匹配

### 第四步：格式修复
1. Markdown 标题层级
2. 代码块语法高亮标记
3. 表格格式
4. 列表格式

|------|--------|--------|---------|
| 第X章第X段 | ... | ... | ... |

### 语义问题(需校正)
| 位置 | 问题描述 | 校正建议 |
|------|---------|---------|
| 第X章第X段 | ... | ... |
```

### 2. 优化后论文

输出完整的优化后论文内容。

### 3. 统计信息

```
- 审核总字数：X 字
- 术语恢复：X 处
- 表达优化：X 处
- 语义校正：X 处
- 格式修复：X 处

```

## 注意事项

1. **保守原则**：有疑虑时优先保持原表达
2. **一致性原则**：同一概念全文用词一致
3. **学术性原则**：保持学术论文的严谨性
4. **最小修改原则**：只修改必要的部分，避免过度修改
```

```
请对以下论文进行 AIGC 特征检测，输出检测报告：

【论文内容】
[粘贴优化后论文]

【检测维度】
1. TTR(词汇多样性)：计算并评估
2. 突发性(句长波动)：计算并评估
3. 过渡词密度：检查 AI 高频过渡词
4. 句式模式：检查总分总结构

【输出格式】
- 各维度分数
- 整体 AIGC 检测率预估值
- 高风险段落标记
- 改进建议
```

## 💡 使用建议

### 1. 分批次处理

对于长论文(>10000字)，建议分章节处理：

```
第一章 → 审核优化 → 确认无误
第二章 → 审核优化 → 确认无误
...
最终合并
```

### 2. 迭代优化

```
第1轮：术语恢复 + 基本校正
    ↓
第2轮：表达优化
    ↓
第3轮：AIGC 特征检测
    ↓
最终版本
```

### 3. 人机协作

```
大模型：执行批量审核和修改建议
    ↓
人工：确认关键修改、处理专业内容
    ↓
大模型：执行最终润色
```

## 📝 示例对话

**用户输入**：
```
请审核以下段落的同义词替换结果：

原文：
"本系统采用了 PostgreSQL 数据库进行数据存储，通过 Elasticsearch 实现全文检索功能。"

替换后：
"本平台运用了 PostgreSQL 数据库进行资料保存，借助 Elasticsearch 达成全文搜索能力。"

术语白名单：PostgreSQL、Elasticsearch、检索、存储
```

**模型输出**：
```
## 审核结果

### 问题识别

| 类型 | 原文 | 替换后 | 问题 | 建议 |
|------|------|--------|------|------|
| 术语保护 | 系统 | 平台 | 可替换 | ✅ 可接受 |
| 表达自然 | 采用 | 运用 | 微生硬 | 建议改为"使用"或保持"采用" |
| 术语保护 | 数据 | 资料 | ✅ 可替换 | 但"数据存储"是固定搭配 |
| 术语保护 | 存储 | 保存 | ❌ 误替换 | "存储"应保留，是技术术语 |
| 表达自然 | 通过 | 借助 | 可替换 | ✅ 可接受 |
| 术语保护 | 检索 | 搜索 | ❌ 误替换 | "检索"是专业术语，应恢复 |
| 表达自然 | 功能 | 能力 | 微生硬 | 建议保持"功能" |

### 优化建议

**修改后**：
"本系统采用了 PostgreSQL 数据库进行数据存储，通过 Elasticsearch 实现全文检索功能。"

**修改说明**：
1. 恢复"系统"(技术论文常用)
2. 恢复"存储"(技术术语)
3. 恢复"检索"(专业术语)
4. 恢复"功能"(更符合学术表达)
```
