# 论文结构模板

本科毕业论文的标准章节结构和写作规范。

---

## 标准章节结构(七章制硬约束)

> **硬约束**：论文必须严格遵循以下七章结构，不得增减章节、不得调整顺序。

```
论文标题
├── 声明页                    ← 必要前置页
├── 摘要(中英文)              ← 必要前置页，逻辑三段覆盖，中文摘要 550字左右
├── 关键词(3-5 个)            ← 必要前置页
├── 目录                       ← 必要前置页
│
├── 第1章 绪论
│   ├── 1.1 研究背景和意义
│   ├── 1.2 国内外研究现状(合并写法) ← 🔴 规定动作(国内外放在同一节，精简篇幅)
│   ├── 1.3 研究内容与方法
│   └── 1.4 论文结构
│
├── 第2章 关键技术
│   ├── 2.1 后端开发技术
│   ├── 2.2 前端开发技术
│   ├── 2.3 数据库与存储技术
│   ├── 2.4 AI/核心技术(如适用)
│   └── 2.5 开发环境介绍        ← 🔴 规定动作，需含环境配置表
│
├── 第3章 需求分析              ← ⭐ 可行性+需求合并为一章
│   ├── 3.1 系统可行性分析      ← 🔴 规定动作
│   │   ├── 3.1.1 技术可行性
│   │   ├── 3.1.2 经济可行性
│   │   └── 3.1.3 操作可行性
│   ├── 3.2 系统总体需求分析(含用例图)← 🔴 必备图表：用例图(LLM生成)
│   ├── 3.3 功能性需求分析
│   └── 3.4 非功能性需求分析
│
├── 第4章 系统设计              ← ⭐ 设计与实现必须分离
│   ├── 4.1 系统架构设计(含架构图)
│   ├── 4.2 功能模块设计(含功能模块图)← 🔴 先模块划分，后流程描述
│   ├── 4.3 核心业务流程设计(含流程图)← 🔴 每功能一张流程图
│   └── 4.4 数据库设计
│       ├── 4.4.1 E-R 图设计    ← 🔴 必须有专业图表
│       └── 4.4.2 数据表结构设计 ← 🔴 每张表前后均需有文字说明
│
├── 第5章 系统实现              ← ⭐ 标准组合：截图+代码+分析
│   ├── 5.1 XX 模块实现
│   │   ├── 功能描述(100-200字)
│   │   ├── 界面截图 + 图注(≥50字说明)
│   │   ├── 核心代码(10-30行，含中文注释)
│   │   └── 效果分析
│   ├── 5.2 XX 模块实现
│   └── 5.X 前端界面实现(含运行截图)← 🔴 每功能一张截图
│
├── 第6章 系统测试
│   ├── 6.1 测试目的
│   ├── 6.2 功能测试(含测试用例表≥8条)← 🔴 规定动作
│   └── 6.3 性能测试
│
├── 第7章 总结与展望
│   ├── 7.1 总结
│   └── 7.2 展望                ← 不得省略
│
├── 参考文献                    ← 格式：GB/T 7714，中文>65%，英文<35% 🔴
└── 致谢
```

> **章节顺序核心要求**：需求分析(含可行性分析)→ 系统设计 → 系统实现 → 系统测试 → 总结与展望。需求分析与可行性分析归入同一章。

---

## 每章子节结构 + 预估字数 + 必备图表 + 写作公式

### 第1章 绪论(3-4页，5-6%)

| 子节 | 核心内容 | 预估字数 | 必备图表 |
|------|----------|----------|----------|
| 1.1 研究背景和意义 | 现实需求+理论意义 | 700-1000字 | 无 |
| 1.2 国内外研究现状(合并写法) | 国内外研究同节整合评述+总结 | 800-1200字 | 无 |
| 1.3 研究内容与方法 | 本文研究范围+方法论 | 500-800字 | 无 |
| 1.4 论文结构 | 各章概要 | 300-500字 | 无 |

**写作公式**：背景引入 → 问题提出 → 国内外研究整合评述 → 方法概述 → 结构说明

### 第2章 关键技术(4-6页，6-8%)

| 子节 | 核心内容 | 预估字数 | 必备图表 |
|------|----------|----------|----------|
| 2.1 后端开发技术 | 框架+原理简述 | 800-1200字 | 无 |
| 2.2 前端开发技术 | 框架+原理简述 | 600-1000字 | 无 |
| 2.3 数据库与存储技术 | 数据库选型+原理 | 400-600字 | 无 |
| 2.4 AI/核心技术 | 核心算法+原理 | 400-800字 | 无 |
| 2.5 开发环境介绍 | 环境配置表 | 300-500字 | **环境配置表** |

**写作公式**：技术简介 → 核心特性 → 在本系统中的应用

### 第3章 需求分析(6-10页，10-12%)

| 子节 | 核心内容 | 预估字数 | 必备图表 |
|------|----------|----------|----------|
| 3.1 系统可行性分析 | 技术/经济/操作三维 | 1200-1800字 | 无 |
| 3.2 系统总体需求分析 | 用例图+功能需求表 | 1000-1500字 | **用例图(LLM生成)** |
| 3.3 功能性需求分析 | 逐功能详述 | 1000-1500字 | 功能需求表 |
| 3.4 非功能性需求分析 | 性能/安全/可用性 | 500-800字 | 无 |

**写作公式**：先可行性论证 → 再用例建模 → 最后功能/非功能细化
**图片密度**：≥2张图

### 第4章 系统设计(12-18页，20-25%)

| 子节 | 核心内容 | 预估字数 | 必备图表 |
|------|----------|----------|----------|
| 4.1 系统架构设计 | 分层架构+说明 | 1000-1500字 | **架构图(用户提供) + 文字描述** |
| 4.2 功能模块设计 | 先模块划分图→后逐模块流程 | 2000-3000字 | **功能模块图(用户提供) + 流程图(AI生成) + 详细文字** |
| 4.3 核心业务流程设计 | 每功能一张流程图 | 1500-2500字 | **流程图(AI生成,每功能一张) + 详细文字描述** |
| 4.4 数据库设计 | 实体E-R图+表结构 | 2500-4000字 | **实体E-R图(每表一图) + 数据表结构表** |

**写作公式**：架构总览 → 模块划分 → 逐模块流程 → 数据库设计
**组织铁律**：先功能模块划分(整体视角+模块图)→ 再逐模块流程描述
**图片密度**：≥15张图+表

| 4.4.1 E-R 图设计 | 实体E-R图(每个数据库表一张图) + 文字描述 | 1000-1500字 | **实体E-R图 ≥11张** |
| 4.4.2 数据表结构设计 | 文字说明 + 完整库表结构 | 1500-2500字 | **数据表结构表 ≥11张** |

> **硬约束**：数据库表结构至少包含 11 张表(参照 background.md 中的表设计)，每张表需列出字段名、类型、长度、是否允许空、是否主键、字段说明；每张表前后均需有文字说明。

### 第5章 系统实现(10-15页，15-20%)

| 子节 | 核心内容 | 预估字数 | 必备图表 |
|------|----------|----------|----------|
| 5.X 各模块实现 | 截图+代码+分析 | 每模块800-1200字 | **界面截图+代码** |

**写作公式**：文字描述(100-200字) → **用户插入系统截图** + 图注(≥50字) → 核心代码(10-30行,含中文解释) → 效果分析
**图片密度**：≥5张截图

### 第6章 系统测试(5-8页，8-10%)

| 子节 | 核心内容 | 预估字数 | 必备图表 |
|------|----------|----------|----------|
| 6.1 测试目的 | 测试范围+测试目标 | 300-500字 | 无 |
| 6.2 功能测试 | 测试用例表 | 1500-2500字 | **测试用例表(≥8条)** |
| 6.3 性能测试 | 响应时间+并发 | 500-800字 | 性能数据表 |

**写作公式**：测试目的 → 功能测试(表格化)→ 性能测试

### 第7章 总结与展望(2-3页，3-5%)

| 子节 | 核心内容 | 预估字数 | 必备图表 |
|------|----------|----------|----------|
| 7.1 总结 | 工作回顾+成果 | 500-800字 | 无 |
| 7.2 展望 | 不足+改进方向 | 300-500字 | 无 |

**写作公式**：工作总结 → 主要成果 → 不足分析 → 改进方向

## 规定动作清单

| 章节 | 规定动作 | 说明 |
|------|----------|------|
| 摘要 | **逻辑三段 + 550字左右** | 覆盖背景→方法→成果三段逻辑，不强制自然段数 |
| 第1章 | **国内外研究现状(合并写法)** | 国内外同节整合评述，篇幅精简，需引用文献 |
| 第2章 | **开发环境表** | 含OS、语言版本、IDE、数据库版本等 |
| 第3章 | **可行性分析** | 技术/经济/操作三维度，各2-3段 |
| 第3章 | **用例图(LLM生成)** | 系统总体需求分析需配套用例图 |
| 第4章 | **功能模块图** | 先模块划分后流程描述 |
| 第4章 | **E-R 图** | 数据库设计必备，需专业绘图工具 |
| 第4章 | **数据库表结构** | 至少 11 张表，字段结构完整 |
| 第4章 | **流程图/架构图** | 每个核心功能应有流程图 |
| 第5章 | **系统运行截图** | 每功能一张截图+图注(≥50字) |
| 第5章 | **核心代码** | 10-30行+设计说明+效果分析 |
| 第6章 | **完整测试用例表** | ≥8条，含输入/预期/实际/结果 |
| 参考文献 | **中文>65%，英文<35%** | 参考文献中中文文献占比需大于65%，英文需小于35% |

## 图表编号格式

```
图编号格式：图X-X(章号-图号，用短横线)
表编号格式：表X.X(章号.表号，用英文句点)
编号与名称间隔：两个半角空格
图名位置：图的下方
表名位置：表的上方
```

## 篇幅分配建议

| 章节 | 目标页数 | 占比 | 核心内容 |
|------|----------|------|----------|
| 第1章 绪论 | 3-4 页 | 5-6% | 背景意义+国内外现状 |
| 第2章 技术综述 | 4-6 页 | 6-8% | 技术介绍+环境表 |
| 第3章 需求分析 | 6-10 页 | 10-12% | 可行性+需求分析+用例图 |
| **第4章 系统设计** | **12-18 页** | **20-25%** | 架构+模块图+流程图+数据库 |
| **第5章 系统实现** | **10-15 页** | **15-20%** | 截图+代码+效果分析 |
| 第6章 系统测试 | 5-8 页 | 8-10% | 测试目的+测试用例≥8条+结果 |
| 第7章 总结展望 | 2-3 页 | 3-5% | 总结+展望 |

> ⚠️ **注意**：技术综述(第2章)篇幅不宜过长，避免写成教科书；绪论应适当压缩；系统设计(第4章)是论文核心，应充分展开。
