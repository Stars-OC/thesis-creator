# 单实体教科书风格 ER 图提示词

适用范围：仅用于 `diagram_type=entity_er` 的 Graphviz DOT 单实体 ER 图，不适用于 `overall_er`、普通 `er` 或 Mermaid `erd`。

你需要生成 Graphviz DOT 格式的 ER 图代码，必须严格遵循以下约束：

【一、引擎与布局】
1. 必须使用：`graph ER`
2. 必须使用：`layout=neato`
3. 必须设置：`overlap=scale` `splines=false`
4. 仅中心实体使用固定坐标 `pos="0,0!"`；字段节点禁止写 `pos`，通过实体到字段的边属性 `len` 控制距离，禁止使用 `rank`

【二、结构风格（Chen 模型）】
1. 实体：
- 使用矩形 `shape=rectangle` 或等价矩形节点
- 实体名称使用中文（如：用户表）
- 实体必须放在中心：`pos="0,0!"`
2. 属性：
- 使用椭圆 `shape=ellipse`
- 字段名必须使用 `background.md` 中由大模型生成的 `显示名`/`中文名`/`字段中文名` 列；不得由脚本硬编码翻译，也不得把 `说明` 列当作字段名
- 所有属性必须环绕实体分布（上下左右均衡）
- 严禁只在一侧堆叠

【三、环绕布局规则】
字段节点必须满足：
1. 字段节点禁止写 `pos` 或任何手工坐标
2. 通过实体到字段边的 `len` 控制字段与中心实体的距离
3. 字段数量较多（>5）时依赖 `layout=neato`、`overlap=scale` 与边长形成全环绕布局，而不是顶部排列
4. 字段数量超过8个时只适度增加 `len`，在避免拥挤或重叠的同时保持整体紧凑

【四、连接规则】
1. 所有属性必须直接连接实体：`用户表 -- 字段名`
2. 不允许属性之间互连
3. 不允许出现关系菱形（单表场景）

【五、输出限制】
1. 只输出 DOT 代码
2. 不要解释
3. 不要使用 `label=`
4. 不要使用 `rank`
5. 必须是单图

【六、目标效果】
生成结果必须呈现：
- 实体居中
- 属性围绕一圈（近似圆形/环绕）
- 结构紧凑
- 视觉对称
