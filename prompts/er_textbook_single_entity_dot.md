# 单实体教科书风格 ER 图提示词

适用范围：仅用于 `diagram_type=entity_er` 的 Graphviz DOT 单实体 ER 图，不适用于 `overall_er`、普通 `er` 或 Mermaid `erd`。

你需要生成 Graphviz DOT 格式的 ER 图代码，必须严格遵循以下约束：

【一、引擎与布局】
1. 必须使用：`graph ER`
2. 必须使用：`layout=neato`
3. 必须设置：`overlap=true` `splines=true`
4. 必须使用 `pos` 精确坐标控制布局，禁止使用 `rank`

【二、结构风格（Chen 模型）】
1. 实体：
- 使用矩形 `shape=rectangle` 或等价矩形节点
- 实体名称使用中文（如：用户表）
- 实体必须放在中心：`pos="0,0!"`
2. 属性：
- 使用椭圆 `shape=ellipse`
- 字段名必须使用中文
- 所有属性必须环绕实体分布（上下左右均衡）
- 严禁只在一侧堆叠

【三、环绕布局规则】
属性坐标必须满足：
- 上方：`y > 0`
- 下方：`y < 0`
- 左侧：`x < 0`
- 右侧：`x > 0`

并且：
1. 至少覆盖上、下、左、右四个方向
2. 左右数量接近，上下数量接近
3. 字段数量较多（>5）时必须使用全环绕布局，而不是顶部排列

【四、连接规则】
1. 所有属性必须直接连接实体：`用户表 -- 字段名`
2. 不允许属性之间互连
3. 不允许出现关系菱形（单表场景）

【五、输出限制】
1. 只输出 DOT 代码
2. 不要解释
3. 不要使用 `label=`
4. 不要使用 `rank`
5. 必须是单图

【六、目标效果】
生成结果必须呈现：
- 实体居中
- 属性围绕一圈（近似圆形/环绕）
- 结构紧凑
- 视觉对称
