# 法学论文写作提示词

## 学科定位

法学论文（含民商法、经济法、刑法、行政法、宪法、诉讼法、国际法、知识产权法、环境与资源保护法、劳动与社会保障法等）以**规范分析 + 案例论证 + 制度完善**为核心论证脉络。论文必须服务于一个**具体的法律问题**，禁止泛泛而谈。

## 章节定位（6 章结构）

| 章节 | 核心任务 | 写作要点 |
|------|---------|----------|
| ch1 引言 | 提出法律问题与研究意义 | 立法/司法实务背景 → 学界争议 → 本文研究问题 → 研究方法（规范分析/案例分析/比较法） |
| ch2 基本理论概述 | 法理基础与概念界定 | 概念界定 → 法律性质 → 制度沿革 → 域内外学说梳理 |
| ch3 现状考察与案例分析 | 论文核心 | 现行立法梳理 + 司法实务现状（含指导性案例/典型判例分析）+ 比较法考察（可选） |
| ch4 问题与原因分析 | 诊断当前制度缺陷 | 立法层面问题 + 司法适用问题 + 根本原因（理论 / 制度 / 配套机制） |
| ch5 完善建议 | 提出对策 | **立法完善建议** + **司法解释或裁判规则建议** + **配套制度建议**，必须有法理依据 |
| ch6 结语 | 总结全文 | 主要结论 + 研究局限 + 后续研究方向 |

## 写作硬约束

### 必有

- **法条引用规范**：《中华人民共和国民法典》第1062条；不得只写"民法典相关规定"
- **案例引用规范**：完整案号 + 审理法院 + 裁判日期；指导性案例标明"指导案例X号"
- **司法解释引用规范**：《最高人民法院关于XX的解释》（法释〔YYYY〕X号）第X条
- **现行有效性**：所有引用法律必须为现行有效版本；已修订条款必须注明修订时间
- **学术中立**：完善建议须在法律框架内，禁止政治倡导

### 禁止

- 出现"系统设计""数据库""接口""算法实现"等计算机学科用语
- 虚构域外法制度或案例
- "相关法律规定""有关司法解释"等空泛表述
- 罗列法条而无解释、无适用、无问题分析

## 图表偏好（可选）

法学论文以文字论证为主，图表非必需。如配图，可选：
- **法条/规范结构图**（PlantUML）：展示某部法律的体系或法律渊源层级
- **案例时序图**（PlantUML）：展示典型案例的事实经过与诉讼程序
- **制度比较表**（Graphviz/普通表格）：中外制度对比、新旧法对比
- **决策树**（Graphviz）：法律适用的构成要件判断逻辑

禁止使用 ER 图、类图、系统架构图。

## 文献偏好

- 中文 ≥85%，英文 ≤15%（除比较法专题外）
- 中文优先：中国法学、法学研究、中外法学、现代法学、政法论坛、法商研究、清华法学、法学评论、比较法研究
- 英文优先：Harvard Law Review、Yale Law Journal、Columbia Law Review、Stanford Law Review、AJIL 等
- **一手材料优先**：法条 > 司法解释 > 指导性案例 > 权威释义 > 学术论文

## AIGC 风险点

- "随着社会的发展""在当前形势下"等开篇套话 → 改为具体立法事件或司法争议引入
- "首先/其次/最后"段落顺承 → 改为"从立法层面看 / 从司法实务看 / 从配套机制看"
- 建议部分机械排比 → 增加针对不同主体（立法机关/司法机关/行政机关/当事人）的差异化讨论
- 案例描述模板化 → 案例事实部分必须聚焦争议焦点，删除与论点无关的细节
