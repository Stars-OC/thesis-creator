# 经济管理论文写作提示词

## 学科定位

经济管理类论文（含工商管理、市场营销、财务管理、人力资源、电子商务、物流管理、旅游管理、公共管理等）以**问题导向 + 案例/数据支撑 + 管理建议**为核心论证脉络，禁止套用计算机学科的"系统设计/实现/测试"章节模板。

## 章节定位（5 章结构）

| 章节 | 核心任务 | 写作要点 |
|------|---------|----------|
| ch1 绪论 | 提出研究问题与意义 | 现实背景 → 理论缺口 → 研究问题 → 研究方法 → 论文框架 |
| ch2 相关理论与文献综述 | 建立理论框架 | 国内外研究现状 → 理论基础 → 分析框架（如 SWOT、五力模型、波特价值链、4C/4P、TAM、SOR） |
| ch3 研究设计 | 说明研究路径 | **实证类**：变量定义、数据来源、模型设定；**案例类**：案例选取理由、资料来源、分析维度 |
| ch4 实证分析或案例分析 | 论文核心 | 描述性统计 + 相关性/回归 + 稳健性检验（实证）；或 案例背景 + 问题诊断 + 成因分析（案例） |
| ch5 研究结论与建议 | 回答研究问题 | 主要结论 → 管理建议（分层）→ 研究局限与展望 |

## 写作硬约束

### 必有

- **第3-4章**：变量定义表、描述性统计表（实证类）或 案例企业基本情况表（案例类）
- **数据来源标注**：所有图表必须在图下方标注"数据来源：xxx"
- **理论框架闭环**：第2章提出的分析框架必须在第4章被实际使用
- **建议可操作**：第5章建议必须按"战略层/管理层/执行层"或"短期/中期/长期"分层

### 禁止

- 出现"系统架构图""数据库表""接口实现""核心代码"等计算机学科用语
- 案例分析章节只罗列事实而不做诊断、不引用理论
- 完善建议出现"应加强 / 应重视 / 应完善"等空泛动词后无具体动作

## 图表偏好

- 第4章每章 ≥3 张图表，类型：**柱状图、折线图、饼图、雷达图、矩阵图、组织架构图、案例时序图**
- 禁止使用 ER 图、类图、系统流程图（与学科特征不符）
- 若涉及流程，使用业务流程图（BPMN 风格的 PlantUML activity diagram）

## 文献偏好

- 中文 ≥70%，英文 30%
- 中文优先：管理世界、中国工业经济、经济研究、南开管理评论、会计研究、营销科学学报
- 英文优先：SMJ / AMJ / JoM / JMR / JCR / MISQ / OS
- 案例类论文允许引用企业年报、上市公司公告、行业研究报告（如艾瑞、艾媒、Wind 数据）作为一手资料

## AIGC 风险点

- "在当今/随着...的发展"开篇模板词 → 删除或改为具体年份和行业事件引入
- "首先/其次/最后"段落顺承 → 改为基于分析逻辑的过渡（如"在描述性统计之后，进一步检验..."）
- 政策建议机械排比 → 增加企业/行业差异化讨论
