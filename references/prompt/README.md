# 论文背景信息（Prompt）

本目录用于存放您的论文背景信息，是 AI 理解您论文需求的核心依据。**这是必需填写的文件。**

## 📁 文件说明

| 文件 | 状态 | 说明 |
|------|------|------|
| `template.md` | 📝 模板 | 背景信息模板，请复制此文件 |
| `background.md` | ✏️ 用户创建 | 您填写的背景信息，**必需** |
| `discussion_log.md` | 🤖 自动生成 | 与 AI 的讨论记录 |

## 🚀 快速开始

### 步骤 1：复制模板

```bash
# 方式一：命令行复制
cp template.md background.md

# 方式二：手动操作
# 在文件管理器中复制 template.md，重命名为 background.md
```

### 步骤 2：填写背景信息

打开 `background.md`，逐项填写各项内容。

> [!IMPORTANT]
> 如果需要自动生成第4章 E-R 图，请优先填写“数据库表设计”区块，并尽量为每张表提供标准 Markdown 表格（字段名、类型、长度、允许空、主键、说明）。脚本会优先按这里的表结构生成“一表一图 + 一图一说明”。

### 步骤 3：启动 Skill

在 Claude Code 中说：

```
帮我写论文，我已经填写了背景信息
```

或直接说：

```
帮我写论文，主题是《XXX》
```

### 步骤 4：与 AI 讨论

AI 将读取您的背景信息，并与您进行不少于三轮的讨论。

## 📝 填写示例

以下是填写完成的 `background.md` 示例：

```markdown
# 论文背景信息

## 基本信息

- **论文题目**：《基于深度学习的电商评论情感分析系统设计与实现》
- **专业方向**：计算机科学与技术
- **姓名**：张三
- **指导教师**：李教授

---

## 一、研究背景

随着电子商务的快速发展，电商平台的用户评论数据呈爆炸式增长。这些评论中蕴含着丰富的用户情感信息，对于商家改进产品和服务具有重要价值。然而，传统的人工分析方法效率低下，难以处理海量评论数据。深度学习技术的发展为自动化情感分析提供了新的解决方案。

本研究旨在设计并实现一个基于深度学习的电商评论情感分析系统，帮助商家快速、准确地了解用户反馈，从而优化产品和服务策略。

---

## 二、研究问题

1. 如何构建高质量的电商评论情感分析数据集？
2. 如何选择和优化适合中文电商评论的情感分析模型？
3. 如何设计一个用户友好的情感分析系统？

---

## 三、研究方法

- [x] 文献研究法 - 分析现有情感分析方法
- [x] 实验研究法 - 对比不同模型效果
- [x] 系统开发法 - 实现完整系统

---

## 四、数据来源

- 数据类型：电商评论文本数据
- 获取方式：京东、淘宝公开评论数据
- 样本规模：约 50,000 条评论

---

## 五、技术方案

- 开发语言：Python
- 框架：PyTorch、Flask
- 数据库：MySQL
- 前端：Vue.js

---

## 六、论文结构规划

第1章：绪论
第2章：相关技术综述
第3章：系统需求分析
第4章：系统设计
第5章：系统实现
第6章：系统测试
第7章：总结与展望

---

## 七、写作要求

- 总字数要求：12,000 字以上
- 查重率上限：≤ 30%
- AIGC 检测率上限：≤ 20%
- 参考文献格式：GB/T 7714-2015
- 特殊要求：需要外文翻译

---

## 八、已有的参考资料

[1] 刘兵. 情感分析：挖掘观点、情感和情绪[M]. 机械工业出版社, 2017.
[2] Devlin J, et al. BERT: Pre-training of Deep Bidirectional Transformers[J]. NAACL, 2019.
```

## ⚠️ 注意事项

1. **填写不完整**
   - 未填写的部分，AI 会主动询问
   - 您也可以选择让 AI 自动推断

2. **讨论过程中可更新**
   - 讨论中如需调整，可直接更新 `background.md`
   - AI 会重新读取最新内容

3. **三轮讨论后确认**
   - 三轮讨论后，AI 会生成「背景确认报告」
   - 请仔细确认报告内容后再进入写作阶段
